# AI x Web3 School Learning Plan / 个人学习计划

## Summary / 总览

This plan is designed for a beginner-friendly developer and Hackathon path with 60-90 minutes per day. The default rhythm is: understand one concept, complete one small artifact, and write one check-in draft.

本计划面向“新手起步 + 开发/Hackathon + 每天 60-90 分钟”的学习节奏。每天默认完成三件事：理解一个概念、产出一个小作品、写一份打卡草稿。

Core references:

- Handbook: https://aiweb3.school/zh/handbook/
- WCB learning page: https://web3career.build/zh/programs/AI-Web3-School#tab=learning
- Learning Agent prompt: https://aiweb3.school/learning-agent.zh.txt

## Daily 60-90 Minute Rhythm / 每日节奏

### Minimum Path / 最小路径, 30-45 min

- Read the assigned WCB task or one Handbook section.
- Write a 5-8 bullet summary in the daily note.
- Record one question, one useful link, and one next action.
- Draft the WCB check-in text locally.

### Recommended Path / 推荐路径, 60-90 min

- Complete the Minimum Path.
- Build or run one tiny experiment, for example a prompt workflow, API sketch, wallet flow diagram, or contract interaction note.
- Create one task note in `tasks/` or one experiment note in `experiments/`.
- Identify whether the Handbook needs feedback.

### Challenge Path / 挑战路径, 90+ min

- Complete the Recommended Path.
- Connect the day to a Hackathon idea: user pain, workflow, data, interface, risk, or demo plan.
- Update `hackathon/` with a product note, prototype sketch, or demo checklist.

## Phase 1: Shared Vocabulary / 阶段一：共同语言

Goal: build enough AI and Web3 vocabulary to follow course material confidently.

- AI basics: prompts, models, context, tools, agents, evaluation.
- Web3 basics: wallets, accounts, chains, transactions, gas, contracts, signatures.
- Output: daily notes, concept glossary, and first blockers list.

## Phase 2: Agent Workflow / 阶段二：Agent 工作流

Goal: understand how an AI agent turns user intent into steps, tool calls, artifacts, and feedback loops.

- Learn: task decomposition, tool use, memory, safety boundaries, evaluation.
- Practice: write agent prompts, inspect inputs/outputs, compare manual vs agent-assisted workflows.
- Output: small workflow experiments in `experiments/`.

## Phase 3: Web3 Tool Use / 阶段三：Web3 工具使用

Goal: connect AI-assisted workflows to Web3 tasks without compromising wallet safety.

- Learn: wallet permissions, message signing, transaction review, contract reads/writes.
- Practice: document a safe wallet interaction flow and prototype a read-only Web3 task first.
- Output: task notes and safety checklists.

## Phase 4: AI x Web3 Bridge / 阶段四：AI x Web3 结合

Goal: turn isolated concepts into a useful AI x Web3 product direction.

- Explore: agent-assisted research, on-chain data analysis, community workflows, identity, governance, creator tools.
- Decide: choose one narrow user problem and define the smallest demo.
- Output: Hackathon idea note, demo flow, and proof-of-work links.

## Phase 5: Hackathon Prototype / 阶段五：Hackathon 原型

Goal: produce a simple demo that can be explained, tested, and improved.

- Define: target user, problem, workflow, success metric.
- Build: small runnable prototype or clickable demo.
- Prepare: README, demo script, screenshots, risks, next steps.
- Submit: final links in `submissions/`.

## Weekly Review / 每周复盘

- What did I learn?
- What did I build?
- What still feels confusing?
- Which Handbook page could be improved?
- What is the smallest useful Hackathon direction emerging?

## Public Safety Reminder / 公开安全提醒

Never commit real private keys, seed phrases, API keys, access tokens, wallet recovery phrases, passwords, unpublished contact details, internal links, or private community content.
